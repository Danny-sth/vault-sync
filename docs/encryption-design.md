# Vault-Sync — E2EE через MCP (zero-knowledge сервер)

## Цель
Волт шифруется так, что:
- на диске VPS лежит только шифротекст;
- по сети передаётся только шифротекст;
- читать/писать содержимое может лишь владелец ключа;
- доступ к блобам авторизуется MCP-токеном.

## Модель
- **Сервер (VPS) — zero-knowledge.** Хранит и отдаёт шифро-блобы. НИКОГДА не
  держит ключ, не расшифровывает, не грепает содержимое. Видит только: список
  путей-идентификаторов, размеры, хэши блобов, seq/tombstone (метаданные синка).
- **Клиенты (Obsidian-устройства + клиент Claude) — держат ключ.** Шифруют перед
  отправкой, расшифровывают после получения.
- **MCP-токен** = авторизация доступа к блобам. **Ключ шифрования** = отдельный
  секрет, на сервер не попадает. Нужны оба: токен (достать блоб) + ключ (прочитать).

## Единый канал — всё через MCP
Синхронизация Obsidian-устройств тоже идёт через ЭТОТ ЖЕ MCP (не отдельный
REST/WS). MCP-инструменты работают с шифро-блобами:
- `list` → пути + хэш блоба + seq + tombstones (метаданные, без содержимого);
- `get(path)` → шифро-блоб;
- `put(path, blob, baseHash, baseSeq)` → приём блоба, optimistic concurrency по
  хэшу блоба (как сейчас в REST upload);
- `delete(path)` → tombstone.
Claude-клиент и плагин используют одни и те же инструменты; разница лишь в том,
что после `get` каждый расшифровывает блоб своим ключом.

## Крипто (без велосипедов)
- Ключ волта `K` = Argon2id(passphrase, salt), 256 бит. Один и тот же на всех
  клиентах, заводится out-of-band (не через сервер).
- Контент файла: AES-256-GCM, случайный nonce на каждое шифрование,
  AAD = относительный путь (привязка блоба к пути, защита от подмены).
- Формат блоба: `magic | version | nonce | ciphertext+tag`.
- Хэш для concurrency = SHA-256(блоб) — сервер считает его вслепую, как сейчас.
- Библиотеки: TS-сторона — `@noble/ciphers` + `@noble/hashes` (audited). Сервер
  крипто НЕ делает.

## Что ломается и переезжает на клиент
- `VaultNoteService` серверный поиск/чтение/title/snippet работает на плейнтексте
  → на шифротексте невозможно. Поиск/чтение содержимого переезжают на клиента
  (расшифровал блобы → ищет локально). Сервер отдаёт только блобы и метаданные.
